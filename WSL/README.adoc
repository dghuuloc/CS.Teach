= WSL

== Getting Started

=== Installation

Run this first in Powershell

[source,bash]
----
wsl -l -v
----

That will show our installed distros and their exact names. Then use one of those names, for examples:

[source,bash]
----
wsl --set-default Ubuntu
----

or 

[source,bash]
----
wsl --set-default Ubuntu-24.04
----

*only if it appears exactly that way* in the list. Microsoft’s WSL command reference says `wsl --set-default <Distribution Name>` sets the default Linux distribution, and the name must match an installed distro.

If` wsl -l -v` shows no installed distros, install one first:

[source,bash]
----
wsl --list --online
wsl --install -d Ubuntu-24.04
----

If `Ubuntu-24.04` is not in the online list on our system, install the available name instead (for example `Ubuntu`). Microsoft notes `wsl --list --online` shows valid installable distro names, and `wsl --install -d <Distro>` installs one.

Also, if what you actually wanted was to make new installs use WSL 2, that is a different command:

[source,bash]
----
wsl --set-default-version 2
----

That sets the *default WSL version*, not the default distro.

* To delete *Ubuntu (Default)* from WSL, run this in PowerShell:

[source,bash]
----
wsl --unregister Ubuntu
----

That removes the *Ubuntu* distro from WSL completely, and *all files*, *packages*, *and settings inside that distro are permanently deleted*. Microsoft documents `wsl --unregister <DistributionName>` for removing a distro, and specifically notes `wsl --unregister Ubuntu` as the example.

You can confirm first with:

[source,bash]
----
wsl -l -v
----

Then make sure the name is exactly `Ubuntu` before running the unregister command. Microsoft notes `wsl -l` / `wsl --list` shows the installed distro names to use with `--unregister`.
